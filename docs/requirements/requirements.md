# Lovers 要件定義書

## 1. プロダクト概要

### 1.1 目的
同棲している2人がそれぞれの月間支出を管理し、月末に合算した上で折半ではなく実際の支払い額に基づいて清算する。どちらがいくら多く払ったのかを明確にすることで、公平な家計管理を実現する。

### 1.2 背景
- 2人の口座が別々になっている
- 月末に食費や光熱費などの計算が煩雑
- 実際にはどちらかが多く支払っていることが多く、その差分を明確にしたい

---

## 2. 機能要件

### 2.1 支出管理
- **支出入力画面**
  - カテゴリを選択
  - Aさんの支払額とBさんの支払額を同時に入力（例：「Aさん5000円、Bさん0円」）
  - 支払日、説明など必要なメタデータを入力

- **テンプレート機能**
  - グループ作成時にテンプレートを使用するか確認
  - 毎月固定で引かれる支出（家賃など）をテンプレートとして登録可能
  - テンプレートは作成後いつでも自由に追加・編集可能

### 2.2 決算機能
- **月単位の集計**
  - 指定月のすべての支出を合計
  - 「Aさんの総支出」「Bさんの総支出」を計算
  - 差分を算出（例：「Aさんが5000円多く払っているので、Bさんから5000円受け取る」）
  - カテゴリごとではなく、最終的な差分のみを表示

- **年単位の集計**
  - 同様に年間での支出と差分を表示

### 2.3 グループ管理
- **グループ作成・招待**
  - グループ設立者が他のユーザーを招待する形式
  - グループは2人までの制限（将来的には増員の可能性あり）
  - ユーザーは複数のグループに属することができる

### 2.4 カテゴリ
- 支出をカテゴリごとに分類する
- （具体的なカテゴリ一覧は別途定義）

## 3. 技術仕様

### 3.1 技術スタック

| レイヤー | 技術 | デプロイ先 |
|---------|------|----------|
| フロントエンド | Vue / Nuxt | Cloudflare Pages または CloudFront |
| バックエンド | Go / Gin | AWS Lambda |
| データベース | PostgreSQL | Supabase |

### 3.2 ユーザー認証
- **認証方式**
  - メールアドレス + パスワード
  - SRP_A（Secure Remote Password）プロトコルを採用
  
- **認証基盤**
  - AWS Cognito（認証・認可サーバー）
  - ログイン後、JWTトークンを取得して API へのアクセス認証に使用

---

## 4. ドメイン要件

- 金額入力時は実際に支払った金額を入力。税区分は考慮不要
- 支出の最小単位は月単位での集計
- 各グループは2人以下の規模
- ユーザーは複数グループへの所属が可能

---

## 5. 実装計画

- **スコープ** 全機能を一気に実装（MVP ではなく フルセット）
- **次ステップ** 別チャットで DB 設計から開始

---

## 6. 備考

- 将来的にはグループメンバー数を2人以上に拡張する可能性あり
- 通知機能など、追加機能の検討は現段階では不要
